# 📦 База данных — структура и соответствие формату МСП (v4.05)

---

# 🧾 TABLE: documents

Основная сущность — субъект МСП (юрлицо или ИП)

| Поле            | Назначение                  | Тип           | Обяз. | Примечание                  |
| --------------- | --------------------------- | ------------- | ----- | --------------------------- |
| id              | Внутренний PK               | INTEGER       | О     | автоинкремент               |
| doc_id          | ID записи/файла             | TEXT          | О     | уникальный идентификатор    |
| registry_date   | Дата актуальности/включения | DATE          | Н     | зависит от выгрузки         |
| include_date    | Дата включения в реестр     | DATE          | О     | ключевое поле               |
| subject_type    | Тип субъекта                | INTEGER / код | О     | 1 — юрлицо, 2 — ИП, 3 - КФХ |
| category        | Категория МСП               | TEXT / код    | О     | микро / малое / среднее     |
| employees_count | ССЧР                        | INTEGER (N9)  | У     | обяз. при subject_type=1    |
| region_code     | Регион РФ                   | TEXT          | Н     | код субъекта РФ             |
| inn             | ИНН                         | TEXT (10–12)  | О     | с валидацией                |
| ogrn            | ОГРН/ОГРНИП                 | TEXT (13–15)  | О     | с валидацией                |
| name            | Наименование / ФИО          | TEXT          | О     | для ИП — ФИО                |

---

# 🏷 TABLE: okveds

ОКВЭД виды деятельности

|Поле|Назначение|Тип|Обяз.|Примечание|
|---|---|---|---|---|
|id|PK|INTEGER|О|авто|
|document_id|FK → documents.id|INTEGER|О|связь 1:N|
|okved_code|Код ОКВЭД|TEXT|О|классификатор|
|okved_name|Наименование|TEXT|Н|описание|
|okved_version|Версия ОКВЭД|TEXT(4)|О|2001 / 2014|
|is_primary|Основной ОКВЭД|BOOL / 0-1|О|признак|

---

# 📄 TABLE: licenses

Лицензии субъекта

|Поле|Назначение|Тип|Обяз.|Примечание|
|---|---|---|---|---|
|id|PK|INTEGER|О|авто|
|document_id|FK → documents.id|INTEGER|О|связь 1:N|
|license_number|Номер лицензии|TEXT(1–50)|Н||
|license_type|Тип лицензии|TEXT(1–255)|Н||

---

# 📦 TABLE: products

Продукция / услуги

|Поле|Назначение|Тип|Обяз.|Примечание|
|---|---|---|---|---|
|id|PK|INTEGER|О|авто|
|document_id|FK → documents.id|INTEGER|О|связь 1:N|
|product_code|Код продукции|TEXT(1–50)|Н||
|product_name|Наименование|TEXT(1–500)|Н||

---

# 🔗 Связи

```text
documents (1)
 ├── okveds (N)
 ├── licenses (N)
 └── products (N)
```

---

# ⚙️ Бизнес-правила

### employees_count

- N(9)
    
- обязательно при `subject_type = 1`
    

### okved_version

- допустимые значения:
    
    - `2001`
        
    - `2014`
        

### inn

- длина 10 или 12
    
- обязательная валидация контрольной суммы
    

### ogrn

- длина 13 или 15
    
- обязательная валидация
    

### name

- юрлицо: строка
    
- ИП: ФИО (рекомендуется разделение на 3 поля)
